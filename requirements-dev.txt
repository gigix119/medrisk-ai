# Development + testing dependencies for MedRisk AI.
# Installs runtime dependencies too, so `pip install -r requirements-dev.txt` is enough for local dev.
# Pinned to the exact versions verified working together (see README "Dependencies").
#
# pytest            - test runner
# pytest-asyncio    - allows async def test functions (session-scoped event loop, see pyproject.toml)
# pytest-cov        - coverage integration for pytest
# httpx             - async HTTP client, used with ASGITransport against the app in tests
# asgi-lifespan     - triggers FastAPI startup/shutdown (lifespan) events inside async tests
# ruff              - linter + formatter
# mypy              - static type checker
#
# requirements-inference.txt (torch/torchvision/numpy/pillow, CPU-only) is pulled in here too:
# tests/conftest.py builds a real (synthetic) model bundle at collection time, before any test
# runs, so the full test suite - not just tests/inference and tests/integration - needs it
# importable. This does NOT pull in requirements-ml.txt's training-only extras (pandas,
# scikit-learn, matplotlib, tensorboard, h5py) - see tests/inference/test_import_isolation.py
# for what medrisk_inference (and therefore this fixture path) is guaranteed not to need.
#
# PyYAML - needed only by medrisk_research.cli (Phase 7's offline research-evaluation CLI,
# see that package's docstring), which tests/integration exercises against the real test
# database. Deliberately NOT added to requirements.txt itself: the live API
# (app/main.py, app/api/*) never imports medrisk_research or yaml, so the production image
# stays exactly as lean as before - this is purely a test/dev-tooling addition.

-r requirements.txt
-r requirements-inference.txt
