# Known limitations

A single, explicit, no-spin list. Every item here is either already documented elsewhere
(linked) or newly identified during the Phase 8 audit — nothing below is hidden in a more
flattering document.

## Clinical / scientific

- **No clinical validation.** This project has never been validated against real clinical
  outcomes, by a pathologist, or by any clinical process. It is not a medical device.
- **No external validation.** The only model in this repository was trained and evaluated
  entirely on synthetic data generated by this same codebase — there is no held-out, truly
  independent dataset.
- **No real (non-synthetic) trained model exists.** `smoke-baseline-cnn:0.0.1-smoke` is the
  only model artifact in this repository, and it is `synthetic_only: true`,
  `eligible_for_demo: false` by its own manifest. See
  [DATA_AND_MODEL_PROVENANCE.md](DATA_AND_MODEL_PROVENANCE.md).
- **Limited dataset scope.** The real dataset this project is designed around (PCam) is
  derived from a single source distribution (Camelyon16, two Dutch hospitals' scanners) —
  see [dataset-card-pcam.md](dataset-card-pcam.md) "Known biases and limitations." No PCam
  data has actually been used in this repository's current artifacts.
- **Patch-level, not patient-level, methodology.** Even a future real PCam-trained model
  would report patch-level metrics, which do not translate directly to per-patient or
  per-slide diagnostic performance (same dataset card).
- **No regulatory assessment.** No FDA/CE/MDR or equivalent process has been performed or
  attempted. This is explicitly out of scope for a portfolio project.

## Security

- **Rate limiting is per-process/in-memory, not distributed.** If this project ever runs
  multiple worker processes or replicas, each enforces its own limit independently — see
  [SECURITY_AUDIT.md](SECURITY_AUDIT.md) and [THREAT_MODEL.md](THREAT_MODEL.md).
- **No account lockout** after repeated failed login attempts (rate limiting reduces, but
  does not eliminate, brute-force risk).
- **No email verification** on registration.
- **No access-token revocation** before natural (short) expiry — only refresh tokens are
  revocable.
- **No formal independent security review or penetration test** has been performed — Phase
  8's [SECURITY_AUDIT.md](SECURITY_AUDIT.md) is a self-administered review, explicitly
  labeled as such.
- **No full Git history secret scan** — only the current working tree is covered by the new
  CI secret-scanning job.
- **One accepted dependency finding**: `torch==2.11.0` has `CVE-2025-3000` (medium severity);
  not exploitable in this codebase (it never calls `torch.jit.script`) — see
  [SECURITY_AUDIT.md](SECURITY_AUDIT.md) finding F-4.
- **No structured/JSON logging** yet (human-readable only).

## Deployment / operations

- **No public deployment exists.** There is no live URL for this project. See
  [DEPLOYMENT.md](DEPLOYMENT.md) for the chosen-but-unexecuted target and the real blocker
  (no production-grade model, not a missing deployment step).
- **No CSP/security headers verified against a live response** — there is no deployed
  origin to test them against yet.
- **No automated database backup schedule** — there is no deployed instance to schedule one
  against (see [DATABASE_RELEASE_AND_ROLLBACK.md](DATABASE_RELEASE_AND_ROLLBACK.md)).
- **No load testing** of the rate limiter, the inference concurrency semaphore, or the API
  generally under realistic concurrent traffic.
- **No browser-automation tool was available in this development environment across Phases 4
  through 8** — manual click-through verification (390px mobile layout, real keyboard
  navigation, visual Grad-CAM rendering) was substituted with React Testing Library coverage
  and direct dev-server HTTP checks during those phases. **Update, 2026-06-26:** a
  Playwright installation became available in this environment and was used for the portfolio
  publication pass (real desktop/mobile screenshots in
  [docs/assets/screenshots/](assets/screenshots/), captured against a locally running instance
  with a throwaway account). This was a one-off manual screenshot session, not an added
  automated visual-regression suite — no Playwright test files were added to `frontend/`, so
  this remains a real gap in *automated* coverage, not merely a documentation update.

## Process

- **Single-maintainer project.** No code review from a second person, no bus-factor
  mitigation beyond documentation.
- **`research/references.yaml` covers 4 entries**, not every method/library used in this
  codebase — see [DATA_AND_MODEL_PROVENANCE.md](DATA_AND_MODEL_PROVENANCE.md) "Known
  provenance gaps."

## What is explicitly *not* a limitation (already handled correctly)

Listed here only to prevent duplicate future work: CORS allowlisting, JWT secret strength
enforcement, EXIF stripping, decompression-bomb guards, non-root Docker images, the
public/authenticated/admin route boundary, and the "no raw patient data is ever stored"
structural guarantee are all already implemented and verified — see
[SECURITY_AUDIT.md](SECURITY_AUDIT.md) "What was checked and found already adequate."
